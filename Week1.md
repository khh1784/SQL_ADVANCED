## 1. 데이터베이스 알아보기

<!-- 데이터베이스와 DBMS에 관해 배우게 된 점을 적어주세요. -->

```
1. 데이터베이스(DB) : 데이터의 집합

2. DBMS
: 데이터베이스를 관리하고 운영하는 소프트웨어
종류) MySQL, 오라클, SQL 서버, MariaDB
발전과정) 종이에 펜으로 기록 -> 컴퓨터에 파일로 저장
분류)
1) 계층형 : 트리 형태
-> 단점 : 처음 구성 완료 후 변경이 까다로움. 구성원 찾는 것이 비효율적.
2) 망형 : 하위 구성원끼리 연결된 유연한 구조
3) 관계형 : 테이블이라는 최소 단위로 구성. 테이블은 하나 이상의 열과 행으로 구성.

3. SQL
: 관계형 데이터베이스에서 사용되는 언어
```

> **확인문제: 다음 소프트웨어 중에서 DBMS가 아닌 것을 모두 고르세요.**

> MySQL / Excel / Oracle / SQL Server / MariaDB

```

```

## 3. 건물을 짓기 위한 설계도: 데이터베이스 모델링

```
1. 데이터베이스 모델링 : 테이블의 구조를 미리 설계하는 개념 

2. 프로젝트 : 현실 세계 업무를 컴퓨터 시스템으로 옮기는 과정. 대규모 소프트웨어 작성하기 위한 전체 과정.

3. 폭포수 모델 : 프로젝트 계획 -> 업무 분석 -> 시스템 설계 -> 프로그램 구현 -> 테스트 -> 유지보수
장점) 진행 단계 명확
단점) 앞 단계로 돌아가기 어려움

4. 데이터베이스 모델링 : 현실의 작업을 테이블로 변경하기 위한 작업

5. 전체 데이터베이스 구상도
1) 데이터 : 단편적 정보
2) 테이블 : 열과 행으로 구성한 표
3) 데이터베이스 : 테이블이 저장되는 저장소
4) DBMS : 데이터베이스 관리 시스템 또는 소프트웨어
5) 열 : 테이블의 세로
6) 열 이름 : 각 열을 구분하기 위한 이름
7) 테이블 형식 : 열에 저장될 데이터 형식
8) 행 : 실질적 진짜 데이터 -> 행의 개수 = 데이터 개수
9) 기본 키(PK) : 각 행 구분하는 유일한 열
10) SQL : 사람과 DBMS가 소통하기 위한 언어 
> **확인문제: 다음은 폭포수 모델의 절차입니다. 차례대로 나열해보세요.**

> 시스템 설계 / 테스트 / 프로그램 구현 / 프로젝트 계획 / 업무 분석 / 유지보수

```
프로젝트 계획 -> 업무 분석 -> 시스템 설계 -> 프로그램 구현 -> 테스트 -> 유지보수 
```
